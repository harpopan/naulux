**✨  Checklist de Vibe Coding**

Este documento describe un enfoque estructurado para el "vibe coding" asegurando que se completen los pasos preparatorios esenciales antes de comenzar el desarrollo. Siguiendo este checklist, podréis mejorar la claridad del proyecto, mantener la calidad del código y facilitar una colaboración eficiente. Cada paso incluye una descripción y enlaces a tutoriales en vídeo para una orientación práctica.

| 1 | Documento de Requisitos del Producto (PRD) |
| ----- | :---- |
| Antes de iniciar el desarrollo de cualquier aplicación, es fundamental contar con un Documento de Requisitos del Producto (PRD) completo. Este documento actúa como un plano director: detalla el propósito, las funcionalidades y las características de la aplicación desde la perspectiva del usuario. La recomendación es articular el concepto de la aplicación en términos sencillos ante un modelo de lenguaje (LLM) y aprovechar sus capacidades para generar un PRD detallado. **🎬  Recursos en vídeo:** [How to Generate Production-Ready PRDs with AI (Then Build the App)](https://www.youtube.com/watch?v=aLMhV2P7eVg) |  |

| 2 | Documento de Diseño |
| ----- | :---- |
| Un Documento de Diseño traduce los requisitos del PRD en especificaciones visuales e interactivas. Implica seleccionar una inspiración de diseño en plataformas como Dribbble o Mobbin, y usar esa inspiración para guiar la creación de un documento de diseño detallado: comparte el enlace con Gemini y pídele que genere el documento de diseño a partir de él. Este documento incluye wireframes, mockups y especificaciones de interfaz de usuario (UI). **🎬  Recursos en vídeo:** [3 More Ways To ACTUALLY Build Beautiful Websites with Cursor IDE](https://www.youtube.com/watch?v=U1_nzjWUpL4) |  |

| 3 | Reglas Técnicas (Definición del Tech Stack) |
| ----- | :---- |
| Definir las reglas técnicas y el stack tecnológico es un paso fundamental que determina las herramientas, frameworks, lenguajes y plataformas que se utilizarán a lo largo del proyecto. Garantiza consistencia, compatibilidad y escalabilidad. Las reglas técnicas deben especificar claramente las tecnologías elegidas y cualquier directriz concreta para su implementación. **🎬  Recursos en vídeo:** [Ultimate Guide to Choosing a Tech Stack](https://www.youtube.com/watch?v=R5BwFCyV9Mk) |  |

| 4 | Archivo Todo en Markdown (todo.md) |
| ----- | :---- |
| Una vez que el PRD, el Documento de Diseño y las Reglas Técnicas están finalizados y colocados en la carpeta fuente del proyecto, el siguiente paso es generar un archivo todo.md. Este archivo, creado por el vibe coder tras analizar los tres documentos fundacionales, desglosa el proyecto en tareas manejables y específicas por funcionalidad. Este enfoque garantiza que el desarrollo avance de forma sistemática, abordando una funcionalidad a la vez. |  |

| 5 | Inicializar Git |
| ----- | :---- |
| El control de versiones es fundamental en cualquier proyecto de desarrollo de software. Inicializar Git en el directorio del proyecto permite rastrear cambios, colaborar con otras personas y, lo más importante, ofrece la posibilidad de volver a versiones anteriores si alguna implementación sale mal. Esto protege frente a posibles errores introducidos durante el proceso de 'vibe coding'. **🎬  Recursos en vídeo:** [Git and GitHub \- 0 Experience to Professional in 1 Tutorial (Part 1\)](https://www.youtube.com/watch?v=hrTQipWp6co&t=6s) |  |

| 6 | Integración con GitHub y CodeRabbit |
| ----- | :---- |
| Una vez que la aplicación funciona correctamente en local, el código debe subirse a GitHub. Para garantizar la calidad, seguridad y mantenibilidad del código, es muy recomendable integrar CodeRabbit. CodeRabbit es una herramienta basada en IA que analiza todo el código en busca de bugs y vulnerabilidades de seguridad, y proporciona revisiones automáticas directamente dentro de tus pull requests. Este paso es clave para asegurarte de que tu código está listo para producción. **🎬  Recursos en vídeo:** [Getting started with CodeRabbit: From Setup through Your First PR](https://www.youtube.com/watch?v=0tuf3tAkRFo) **CodeRabbit:** [https://www.coderabbit.ai](https://www.coderabbit.ai) |  |
