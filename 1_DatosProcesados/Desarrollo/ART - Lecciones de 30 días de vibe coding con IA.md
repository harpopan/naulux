# Lecciones de 30 días de vibe coding con IA

> [!summary] Resumen
> Experiencia de una persona sin formación tradicional como desarrollador que pasó 30 días programando con herramientas de IA y construyó un producto completo con chat, embeddings, generación de imágenes y ~70.000 líneas de código. Se extraen lecciones prácticas sobre cómo trabajar eficazmente con IA.

Esta es una de las señales más claras de que el desarrollo de software está cambiando rápido.

Un chico sin formación tradicional como desarrollador pasó 30 días haciendo *vibe coding* con herramientas de IA…

Y acabó construyendo:
- Un producto interno de IA
- Funcionalidades de chat + embeddings
- Generación de imágenes
- Sistemas de inicio de sesión
- Infraestructura de pruebas
- Integraciones de analíticas
- Casi 70.000 líneas de código

Pero la parte más interesante no fue el producto.

Fueron las lecciones que aprendió mientras construía con IA:

- Empieza con un PRD (documento de requisitos del producto)
- No construyas todo en una sola sesión
- Construye primero la UI en nuevas funcionalidades
- Enseña a la IA a escribir pruebas, unitarias y e2e
- Usa Git para control de versiones
- Escribe un archivo CLAUDE.md / AGENTS.md adecuado
- Divide los proyectos en tareas diminutas
- Nunca hagas funcionalidades grandes de una sola vez
- Usa Git desde el principio
- Haz que la IA escriba las pruebas
- Crea reglas CLAUDE.md / AGENTS.md para mantener la consistencia

Por eso el debate de «la IA reemplazará a los desarrolladores» se está volviendo complicado.
